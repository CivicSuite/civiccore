CivicCore

Shared platform package for the open-source CivicSuite municipal operations suite — auth, audit, LLM abstraction, ingestion, search, connectors, notifications. MIT-free zone — Apache 2.0 only.

What it is

  • The foundation layer every CivicSuite module (CivicRecords, future modules) depends on.
  • A pip-installable Python package, versioned with strict SemVer against a documented public API surface.
  • Pluggable LLM abstraction targeting local-first deployment via Ollama, with no vendor lock-in.

Install

pip install civiccore  # not yet published — clone + pip install -e .[dev]

Until the first release on PyPI, install from source: 

git clone https://github.com/scottconverse/civiccore.git
cd civiccore
pip install -e .[dev]

Status

v0.1.0.dev0 — Phase 0 scaffold. Package skeleton only. No functional code yet. The 14 subsystems listed in Appendix B of the extraction spec exist as empty subpackages awaiting Phase 1 (shared models + audit chain) per spec section 12. Do not depend on this in production.

Public API surface

The complete list of public symbols, with stability guarantees, is maintained in CivicCore Extraction Spec, Appendix A. Anything not in Appendix A is internal and subject to change without notice.

Compatibility

The version compatibility matrix — which CivicCore versions work with which CivicSuite module versions — lives in the umbrella repository at docs/compatibility/.
