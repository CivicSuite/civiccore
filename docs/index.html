CivicCore

Shared platform library for the open-source CivicSuite municipal operations suite. CivicCore is consumed by CivicSuite modules (records-ai, future modules) — it is not an end-user municipal app. Apache 2.0.

What it is

  • A shared Python library every CivicSuite module depends on. Not an end-user product.
  • Distributed as versioned GitHub release wheels, versioned with strict SemVer against a documented public API surface.
  • Pluggable LLM abstraction targeting local-first deployment via Ollama, with no vendor lock-in.
  • Migration runner, prompt template engine with 3-step override resolver, and model registry — designed to be consumed by downstream modules without tight coupling.

Shipped in v0.2.0

  • civiccore.migrations — migration runner + idempotent guards + civiccore_0001_baseline_v1 + civiccore_0002_llm.
  • civiccore.db — shared SQLAlchemy declarative Base.
  • civiccore.llm — providers (Ollama / OpenAI / Anthropic), templates, model registry, context utilities, structured output.

Install

From the GitHub release wheel (canonical distribution — civiccore is GitHub-wheel-only, not on PyPI):

pip install https://github.com/CivicSuite/civiccore/releases/download/v0.2.0/civiccore-0.2.0-py3-none-any.whl

For development from a clone:

git clone https://github.com/CivicSuite/civiccore.git
cd civiccore
pip install -e .[dev]

Each release publishes SHA256SUMS.txt alongside the wheel and sdist. Verify the checksum before promoting an artifact downstream.

Status

v0.2.0 — Phase 2 shipped. Ships the civiccore.llm module: provider abstraction (Ollama / OpenAI / Anthropic with ABC + factory), prompt template engine with 3-step override resolver, model registry service + admin router, context utilities with prompt-injection defense, and a Pydantic-validated structured-output helper. v0.1.0 was the Phase 1 baseline (migration runner + idempotent guards + civiccore_0001_baseline_v1 shared schema).

Public API overview

The full surface lives in civiccore.llm.__init__ and the CivicCore Extraction Spec Appendix A. High-level shape:

  • Migrations — civiccore.migrations runner + idempotent civiccore_0001_baseline_v1 for downstream alembic chains.
  • db.Base — shared SQLAlchemy declarative base.
  • LLM providers — LLMProvider ABC, register_provider, get_provider, list_providers, build_provider, plus Ollama / OpenAI / Anthropic built-ins.
  • Prompt templates — PromptTemplate ORM + Pydantic schemas, render_template, resolve_template (3-step: app DB override → code-level override → civiccore default).
  • Model registry — ModelRegistry ORM + model_registry_router, get_active_model, require_active_model.
  • Context utilities — TokenBudget, ContextBlock, assemble_context, blocks_to_prompt, sanitize_for_llm.
  • Structured output — StructuredOutput, StructuredOutputFailure, retry-on-malformed.

Compatibility

civicrecords-ai v1.4.0 consumes civiccore v0.2.0. Every CivicSuite module's README declares a CivicCore version range. The suite-wide compatibility matrix lives at CivicSuite/civicsuite/docs/compatibility/.
